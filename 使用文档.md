# Pro-Components 架构级业务组件库指南

## 🌟 设计哲学 (Philosophy)

`Pro-Components` 是基于 Element Plus 封装的企业级高阶组件库，旨在解决中后台系统中大量的重复性 CRUD 开发工作。

我们的核心设计理念是 **Schema-Driven (配置驱动)**：
将庞杂的 UI 模板代码抽象为高度一致的 JSON Schema，让开发者聚焦于**业务数据模型**，而非**DOM 结构**。同时，底层内置了极致的性能优化（浅层响应式）与健壮的网络竞态防护（AbortController），保障企业级应用的绝对稳定。

包含三大核心组件，俗称“CRUD 三剑客”：
1. **`ProForm`**: 基于 Schema 的高级表单，支持栅格布局与联动。
2. **`ProTable`**: 集成高级检索、分页、操作列、防并发请求于一体的数据表格。
3. **`ProDialogForm`**: 弹窗表单，封装了无缝的数据回填与 Loading 状态管理。

## 🛠️ 1. 核心概念：字段模型 (Schema)

无论是表格还是表单，所有的渲染逻辑都由 `FieldSchema` 驱动。得益于强大的 TypeScript 类型推导，你在编写 Schema 时将获得极致的代码提示体验。

### 基础 Schema 结构

```typescript
import type { TableColumnSchema } from '@convenient-ui/types';

const columns: TableColumnSchema[] = [
{
prop: 'username',     // 字段标识 (后端交互的 key)
label: '用户姓名',     // 表头/表单项的名称
type: 'input',        // 组件类型 (自动推导 componentProps)
search: true,         // 是否在 ProTable 的高级检索区域生成搜索框
searchOrder: 1,       // 搜索框的排序

    // 🔥 当 type='input' 时，这里会自动提示 InputProps 的所有属性！
    componentProps: {
      clearable: true,
      maxlength: 20
    },

    // 表单联动逻辑：当某条件满足时才显示该字段
    ifShow: (model) => model.role === 'admin'
}
];
```

## 🚀 2. 核心组件：ProTable

`ProTable` 是使用频率最高的组件，它会自动读取 Schema 中的 `search` 配置生成搜索区域，并接管网络请求、分页和加载状态。

### 基础用法

```html
<template>
  <div style="height: 100vh;">
    <ProTable
      ref="proTableRef"
      :columns="columns"
      :request-api="getUserList"
    >
      <template #toolButton>
        <el-button type="primary" @click="openAddDialog">新增用户</el-button>
        <el-button type="warning" @click="exportData">导出</el-button>
      </template>

      <template #status="{ row }">
        <el-tag :type="row.status === 1 ? 'success' : 'danger'">
          {{ row.status === 1 ? '启用' : '禁用' }}
        </el-tag>
      </template>
    </ProTable>
  </div>
</template>
```
```typescript
<script setup lang="ts">
import { ref } from 'vue';
import { ProTable } from '@chema-ui/vue-ui';
import type { TableColumnSchema } from '@convenient-ui/types';
import { getUserList, deleteUser } from '@/api/user';

// 1. 定义表格列与操作
const columns: TableColumnSchema[] = [
  { type: 'selection', width: 60 },
  { type: 'index', label: '序号', width: 80 },
  { prop: 'username', label: '用户名', type: 'input', search: true },
  { 
    prop: 'role', 
    label: '角色', 
    type: 'select', 
    search: true,
    componentProps: {
      options: [
        { label: '管理员', value: 'admin' },
        { label: '普通用户', value: 'user' }
      ]
    }
  },
  { prop: 'status', label: '状态', type: 'slot' }, // 使用 slot 自定义渲染
  
  // 🔥 强类型的操作列配置
  {
    type: 'action',
    label: '操作',
    width: 200,
    actions: [
      {
        label: '编辑',
        type: 'primary',
        onClick: (row) => openEditDialog(row)
      },
      {
        label: '删除',
        type: 'danger',
        isConfirm: true,             // 开启二次确认气泡
        confirmText: '确定要删除该用户吗？',
        ifShow: (row) => row.role !== 'admin', // 动态显隐：管理员不可删除
        onClick: async (row) => {
          await deleteUser(row.id);
          proTableRef.value?.getTableList(); // 刷新表格
        }
      }
    ]
  }
];

const proTableRef = ref();
</script>
```

### 💡 ProTable 架构亮点
1. **自动防竞态 (Anti-Race Condition)**：底层使用 AbortController 自动取消旧请求，杜绝数据覆盖。
2. **浅层响应式 (Shallow Ref)**：tableData 使用 shallowRef 封装，大幅降低大数据量下的内存与 CPU 损耗。

## 📝 3. 核心组件：ProDialogForm

```html
<template>
<ProDialogForm
ref="dialogFormRef"
:columns="formColumns"
width="500px"
@submit="handleSubmit"
/>
</template>
```
```typescript
<script setup lang="ts">
import { ref } from 'vue';
import { ProDialogForm } from '@chema-ui/vue-ui';
import { saveUser } from '@/api/user';

const dialogFormRef = ref();
const formColumns = [
  { prop: 'username', label: '用户名', type: 'input', rules: [{ required: true }] },
  { prop: 'password', label: '密码', type: 'input', ifShow: (m) => !m.id }
];

const openAddDialog = () => dialogFormRef.value?.open({ title: '新增用户' });

const handleSubmit = async (formData, { close, stopLoading }) => {
  try {
    await saveUser(formData);
    close();
  } catch (error) {
    stopLoading();
  }
};
</script>
```

## 🔧 4. 高阶进阶技巧

### 1. 复杂参数序列化 (Serializer)
```typescript
{
prop: 'dateRange',
label: '创建时间',
type: 'date-picker',
componentProps: { type: 'daterange' },
search: true,
serializer: (val) => ({ startDate: val[0], endDate: val[1] })
}
```

### 2. VNode 渲染函数 (Render)
```typescript
{
prop: 'progress',
label: '进度',
render: ({ row }) => <el-progress percentage={row.progress} />
}
```

## 1. 核心 Schema 配置 (Field & Column)

这是驱动所有组件的核心配置项。`TableColumnSchema` 继承自 `FieldSchema`。

### FieldSchema (通用字段配置)
用于 `ProForm` 和 `ProDialogForm`。

| 属性名 | 类型 | 说明 |
| :--- | :--- | :--- |
| **prop** | `string` | **必填**。字段标识，对应后端接口的 key |
| **label** | `string` | **必填**。表单标签名称 |
| `type` | `ComponentType` | 组件类型（`input`, `select`, `date-picker`, `slot` 等） |
| `defaultValue` | `any` | 字段初始默认值 |
| `componentProps` | `object` | 传递给底层组件的 Props（如 `placeholder`, `disabled` 等） |
| `ifShow` | `(model) => boolean` | 动态显隐逻辑。接收当前表单全量数据，返回布尔值 |
| `rules` | `Rule[]` | 校验规则，同 Element Plus 标准 |
| `colProps` | `object` | 栅格布局配置。如 `{ span: 12, md: 8 }` |
| `fieldStyle` | `object` | 直接作用于组件的样式对象 |
| `className` | `string` | 自定义表单项最外层的类名 |
| `serializer**` | `(val) => object` | **高级**。搜索或提交前的值转换器，可将一个字段拆分为多个 |

### TableColumnSchema (表格特有配置)
在继承上述属性的基础上，增加以下表格特有项：

| 属性名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `search` | `boolean` | 是否将此字段展示在 ProTable 的搜索表单区域 |
| `searchOrder` | `number` | 搜索项的显示顺序，数值越小越靠前 |
| `width` / `minWidth` | `number \| string` | 列宽度 / 最小宽度 |
| `fixed` | `'left' \| 'right'` | 列固定方向 |
| `align` | `'left' \| 'center' \| 'right'` | 单元格对齐方式，默认 `center` |
| `sortable` | `boolean \| 'custom'` | 排序方式，`custom` 为后端分页排序 |
| `actions` | `ActionSchema[]` | 当 `type='action'` 时有效，定义操作按钮数组 |
| `render` | `(scope) => VNode` | 自定义渲染函数（JSX/TSX），优先级高于 slot |
| `headerRender` | `(scope) => VNode` | 自定义表头渲染函数 |

## 2. ProTable 组件 API

### Props
| 参数 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| **columns** | `TableColumnSchema[]` | `[]` | **必填**。列与搜索配置 |
| `requestApi` | `(params) => Promise` | - | 异步请求接口。需透传 `signal` 以支持竞态控制 |
| `dataCallback` | `(res) => {list, total}` | - | 接口返回数据预处理回调 |
| `autoLoad` | `boolean` | `true` | 组件挂载后是否立即执行一次请求 |
| `initParam` | `object` | `{}` | 固定的初始请求参数，不会被搜索重置 |
| `data` | `any[]` | - | 静态数据，不传 `requestApi` 时使用 |
| `searchLabelWidth` | `string` | `'80px'` | 搜索表单的标签宽度 |
| `searchActionColProps` | `object \| number` | `{...}` | 搜索按钮组的布局配置 |

### Slots
| 插槽名 | 说明 |
| :--- | :--- |
| `tableHeader` | 表格上方左侧区域（常用于批量删除、新增按钮） |
| `toolButton` | 表格上方右侧区域（常用于导出、列设置） |
| `empty` | 暂无数据时的占位内容 |
| `[prop]` | 动态插槽。当 Schema 中 `type: 'slot'` 时，通过 `prop` 名匹配 |

### Methods (Expose)
| 方法名 | 说明 |
| :--- | :--- |
| `getTableList` | 刷新表格（保留当前搜索条件） |
| `searchParam` | 获取当前搜索区域的所有参数 |
| `clearSelection` | 清空多选状态 |
| `element` | 获取原生 `el-table` 的实例引用 |

## 3. ProForm 组件 API

### Props
| 参数 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| **columns** | `FieldSchema[]` | `[]` | **必填**。表单字段定义 |
| `modelValue` | `object` | `{}` | 表单数据对象，支持 `v-model` |
| `labelWidth` | `string` | `'100px'` | 标签宽度 |
| `hideAction` | `boolean` | `false` | 是否隐藏自带的“提交/重置”按钮组 |
| `actionColProps` | `object \| number` | `24` | 按钮组的布局配置 |

### Events
| 事件名 | 参数 | 说明 |
| :--- | :--- | :--- |
| `submit` | `(data)` | 点击提交并通过校验后触发，返回深拷贝后的纯净数据 |
| `reset` | - | 点击重置按钮后触发 |
| `update:modelValue`| `(data)` | 表单数据变化时触发 |

### Methods (Expose)
| 方法名 | 说明 |
| :--- | :--- |
| `submit` | 手动触发校验并提交 |
| `reset` | 手动重置表单至初始快照状态 |
| `formRef` | 获取原生 `el-form` 实例 |

## 4. ProDialogForm 组件 API

### Props
| 参数 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| **columns** | `FieldSchema[]` | `[]` | **必填**。弹窗内的表单配置 |
| `title` | `string` | `'表单'` | 默认弹窗标题（可被 open 方法覆盖） |
| `width` | `string` | `'60%'` | 弹窗宽度 |
| `labelWidth` | `string` | `'100px'` | 内部表单的标签宽度 |

### Events
| 事件名 | 参数 | 说明 |
| :--- | :--- | :--- |
| `submit` | `(data, {close, stopLoading})` | 表单提交回调。提供关闭函数和停止按钮 Loading 的函数 |

### Methods (Expose)
| 方法名 | 参数 | 说明 |
| :--- | :--- | :--- |
| `open` | `{ title?: string, row?: object }` | 打开弹窗。`row` 为编辑时的回填数据，已做深拷贝隔离 |
| `close` | - | 关闭弹窗 |
| `stopLoading`| - | 手动停止提交按钮的 Loading 状态 |

## 5. 操作列 ActionSchema 配置

当 `TableColumnSchema` 的 `type` 为 `'action'` 时，`actions` 数组的配置项如下：

| 属性名 | 类型 | 说明 |
| :--- | :--- | :--- |
| **label** | `string` | 按钮显示的文本 |
| **onClick** | `(row) => void` | 点击后的回调函数，参数为当前行原始数据 |
| `type` | `string` | 按钮颜色类型（Element Plus 标准） |
| `ifShow` | `(row) => boolean` | 动态判断当前行的该操作按钮是否显示 |
| `isConfirm` | `boolean` | 是否需要开启二次确认气泡 |
| `confirmText` | `string` | 二次确认气泡的提示文字 |
| `auth` | `string \| string[]` | 权限标识（需配合权限指令/逻辑使用） |
